人脸表情识别系统 — 三种架构方案

技术栈: Python + Streamlit | 输入: 图片/视频/实时摄像头 | 输出: CSV + 统计图表

核心权衡维度

⚡ 实时速度
摄像头≥15fps 算流畅
🎯 识别准确率
展示效果的硬指标
🛡️ 稳定性
不同光照/角度都能用
📦 复杂度
开发量/依赖项
A

轻量快速型 — 极简优先

OpenCV Haar Cascade 人脸检测 + 轻量 Mini-Xception 表情模型 + 基础 Streamlit 页面

优点

  • 依赖极少，安装简单
  • CPU 推理飞快 (30fps+)
  • 代码量最少

缺点

  • Haar Cascade 对侧脸/遮挡敏感
  • 轻量模型准确率约 55-60%
  • 光照变化容易检测不到脸
B

平衡推荐型 — 展示首选 ⭐

MediaPipe Face Detection + HuggingFace ResNet50 FER2013 预训练模型 + 仪表盘式 Streamlit

优点

  • MediaPipe 鲁棒性强，各种角度光照都能检测
  • 深度模型准确率约 65-72%，展示效果不错
  • 速度够快，实时摄像头约 15-20fps
  • Streamlit 仪表盘显得专业

缺点

  • 依赖稍多 (mediapipe, transformers, torch)
  • 首次加载模型需要几秒
  • GPU 可加速但不是必须
C

高精度型 — 追求极致

RetinaFace 人脸检测 + 集成多模型投票 + 全功能 Streamlit 界面

优点

  • 检测最准，侧脸/小脸都能捕获
  • 多模型投票可能到 73-76% 准确率
  • 界面功能丰富

缺点

  • RetinaFace 推理慢 (CPU 上 ~2s/图)
  • 实时摄像头很难跑流畅
  • 依赖重、代码量大、调试困难
  • 课堂展示卡顿影响观感
